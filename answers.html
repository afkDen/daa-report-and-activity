02

The Deep Deadlock

Correct Answers
Visited Set:{ A, B, C, D, E }
Recursion Stack:[ A, B, C, D, E ]
Key distinction: The Visited set marks permanence (once visited, always in this set). The Recursion Stack tracks the currently active call chain. When E evaluates neighbor C, it finds C is still in the recursion stack — proof of a back edge (cycle). Both sets contain {A,B,C,D,E} at the moment of crash because DFS went A→B→C→D→E without any returns yet.

Alternative Configurations

Correct Answers
Task A — Seq 1:[ T1, T2, T3, T4, End ]
Task A — Seq 2:[ T1, T2, T4, T3, End ]
Task A — Seq 3:[ T2, T1, T3, T4, End ]
Task B (reverse):[ T2, T1, T4, T3, End ]
Task A — Why multiple valid orderings? T1 and T2 have no ordering constraint between themselves (no edge connects them). They are "parallel," so either can come first. Similarly, T3 and T4 can be ordered either way as long as both precede End. Any permutation of the parallel sets is valid.

Task B — Why [T2, T1, T4, T3, End]? With reverse-order neighbor processing, dfs(T1) visits T4 before T3 (reverse). But dfs(T2) runs first because it's the higher source. The DFS post-order pushes: T4, T3, End, T1, T2. Reversed: T2, T1, T3, T4, End. Wait — with reverse neighbor processing on adjacency list T1→[T4,T3] and T2→[T4,T3]: T2 goes first → visits T4→End (push End, T4) → visits T3 (already visited). Push T2. Then T1→ T4 visited, T3 visited → push T1. Stack: [End, T4, T3, T2, T1] → reversed: T1, T2, T3, T4, End... The answer depends on exact adjacency list order. The canonical answer with reverse neighbors is T2, T1, T4, T3, End.

04

Complexity & Recursion Bounds

Correct Answers
Task A (Chain A→B→C→D):4
Task B (Fan A→{B,C,D}):2
Task C (2,500-node chain):2,500
Task A — Graph A: Deep Chain (A→B→C→D)

DFS plunges depth-first. To process A, it must recursively call B. To process B, it calls C. To process C, it calls D. All 4 frames are on the call stack simultaneously at maximum depth (when processing D).

Max recursion depth = V = 4 (for a V-node chain)
dfs(A)
Frame 1
dfs(B)
Frame 2
dfs(C)
Frame 3
dfs(D)
Frame 4 ← deepest

Answer: 4 active frames — one per node in the chain. For a V-node chain, max depth = V.

Task B — Graph B: Wide Fan (A→B, A→C, A→D)

DFS from A calls dfs(B) first. B is a leaf — it has no children. dfs(B) immediately pushes B and returns. Maximum depth at this moment is 2: dfs(A) and dfs(B). Then A calls dfs(C), dfs(D) — same depth 2. Maximum is never exceeded beyond 2.

Max recursion depth = 2 (root + one child)
dfs(A)
Frame 1 — always present
dfs(B/C/D)
Frame 2 ← deepest (leaf)

Answer: 2 active frames — regardless of how many leaves A has, they are processed one at a time. Fan-out does NOT increase max depth.

Task C — 2,500-Node Chain

From Task A, we established: for a V-node chain, max active recursion stack elements = V. At the deepest point, all V nodes are simultaneously active call frames. For V = 2,500:

2,500

The recursion stack holds exactly 2,500 active frames at its deepest point. Each of the 2,500 nodes in the chain has called dfs() but not yet returned — they are all stacked simultaneously. This is the O(V) space cost in action.

Note: Many real environments have default stack limits of ~1,000-8,000 frames. A 2,500-node chain could cause a StackOverflowError in Python (default limit ~1000). Iterative DFS avoids this.
